# CS-300-DSA-Analysis-and-Design
Repository for CS 300 DSA Analysis and Design

This repository contains selected artifacts from two major course projects demonstrating my skills in data structures, algorithm analysis, and software design. It also includes two versions of my project reflection: one written for academic assessment and one tailored for future employers reviewing my portfolio.

1. What was the problem you were solving in the projects for this course?
The projects focused on two core problems: evaluating the efficiency of data structures and implementing algorithms to organize and present information. Project One required analyzing the runtime and memory characteristics of several data structures to understand how algorithmic complexity affects performance. Project Two involved designing a program that sorts and prints a list of Computer Science courses in alphanumeric order, demonstrating practical application of sorting algorithms and structured data handling.
2. How did you approach the problem? Consider why data structures are important to understand.
My approach centered on selecting data structures based on their operational characteristics and performance trade-offs. I compared structures such as vectors, hash tables, and binary search trees to determine which best supported the required operations. Understanding these differences was essential because data structures directly influence efficiency, scalability, and clarity of implementation.
3. How did you overcome any roadblocks you encountered while going through the activities or project?
I encountered challenges related to debugging sorting logic, handling edge cases, and validating memory usage. To overcome these issues, I broke the problems into smaller components, tested each part independently, and reviewed algorithmic behavior step-by-step. This iterative debugging process helped clarify misunderstandings and strengthened my ability to reason about program behavior.
4. How has your work on this project expanded your approach to designing software and developing programs?
These projects expanded my approach to software design by emphasizing the value of planning and analysis before writing code. Evaluating data structures and algorithms early in the design process leads to more maintainable and efficient solutions. I gained a deeper appreciation for designing systems that balance performance, clarity, and adaptability.
5. How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?
My work on these projects improved the way I write maintainable and readable code. I now prioritize modular design, consistent naming, clear documentation, and logical structure. These practices make the code easier to understand, extend, and reuse—both for myself and for anyone who may work with it in the future.
